Entity Types & Their Relationships

How Relationships Are Stored

Each entity is a YAML file. Relationships are ID-reference lists on the model.

data/concepts/tool_use.yaml
id: tool_use
name: Tool Use
description: Enabling the model to call
  external functions and APIs
supports:
  - structured_data_extraction
  - agentic_workflows
prerequisites:
  - api_authentication
  - system_prompt
data/assets/tool_use_guide.yaml
id: tool_use_guide
name: Tool Use Facilitator Guide
asset_type: guide
teaches:
  - tool_use
  - system_prompt
demonstrates:
  - structured_data_extraction
supports_tasks:
  - configure_tool_definitions
evidence_links:
  - anthropic_tool_use_docs
  - anthropic_api_reference
No separate edge table. Relationships live on the entities that own them. The KnowledgeGraph class builds a reverse index on load — computed, not stored — so you can traverse in both directions.

The KnowledgeGraph Class

class KnowledgeGraph:
    entities: dict[str, Entity]       # all entities by ID
    _forward: dict[str, list[Edge]]   # id → outgoing edges
    _reverse: dict[str, list[Edge]]   # id → incoming edges

    def load(self, data_dir: str):
        """Load all YAML files, scan relationship fields, build indexes."""

    def get(self, entity_id: str) → Entity:
        """Retrieve an entity by ID."""

    def dependencies(self, entity_id: str) → list[Entity]:
        """What does this entity depend on? (forward links)"""

    def dependents(self, entity_id: str) → list[Entity]:
        """What depends on this entity? (reverse links)"""

    def impact_of(self, entity_id: str) → list[Entity]:
        """Full blast radius — walk reverse transitively."""

    def subgraph(self, entity_ids: list[str]) → list[Entity]:
        """Resolve all connected entities for asset generation."""

Drift Traversal: Walking the Graph

When context_window_docs changes, impact_of() traces the blast radius:

context_window_docs
Evidence changed — start here, follow forward
→ verifies → Context Window
Forward: evidence.verifies field → concept ID
← taught_by ← Prompt Eng Guide
Reverse index: which assets have "context_window" in their teaches list?
← taught_by ← Context Window Reference
Reverse index: same lookup, second result
→ supports → Long Doc Processing
Forward: concept.supports field → capability ID
← demonstrated_by ← Long Doc Demo
Reverse index: which assets have this capability in demonstrates?
The pattern: go forward from the changed entity to find affected concepts/capabilities, then go reverse to find every asset that touches them. Blue = forward (following model fields). Red = reverse (using computed index). Cyan = affected assets.
